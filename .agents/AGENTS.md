# Правила для Antigravity

- **Автоматический деплой через Git**: После завершения любых изменений в коде в рамках выполнения задачи, обязательно автоматически индексируйте, коммитьте изменения с понятным описанием и пушьте их в ветку `main` удаленного репозитория (`git push origin main` или `git push origin master:main`), чтобы сработал автоматический Continuous Deployment (CD). Не требуйте для этого ручного одобрения пользователя.
- **Согласование ТЗ перед изменениями**: Перед внесением изменений в код обязательно напишите, как вы поняли задачу (в виде ТЗ/списка изменений), и вносите правки в файлы и отправляйте на деплой только после явного одобрения («ок») пользователя.

## Критический чек-лист стабильности Mini App

При выполнении любых доработок и рефакторинга в проекте, агент ОБЯЗАН гарантировать, что следующие функции работают и не сломаны:

1. **Автоматический вызов календарика (модалки выбора дат)**:
   * При открытии модального окна переноса даты (`openShiftModal`) или при создании/редактировании шаблона задачи, календарик (нативный выбор даты на мобильных устройствах) обязан открываться автоматически. 
   * Реализация должна использовать таймаут с вызовом `input.showPicker()` и фоллбэком на `input.click()`.

2. **Запрет на блокирующие экраны (секундомеры) при действиях**:
   * Функции `showLoadingOverlay()` and `hideLoadingOverlay()` в `app.js` должны быть пустыми заглушками (`return;`) для всех внутриигровых сетевых запросов. 
   * Никакие действия пользователя (выполнение, пропуск, удаление, взятие задачи, переходы по табам) не должны блокировать интерфейс темной плашкой с секундомером.

3. **Мгновенный переход к деталям (0 секунд)**:
   * Модальное окно деталей задачи (`openChoreDetails` в `app.js`) обязано открываться моментально и синхронно при клике по карточке задачи. 
   * Запрос истории выполнения задачи (`/api/chores/templates/{id}/history`) должен идти в фоновом бесшумном режиме (`silent = true`). Внутри модалки при этом должен отображаться локальный спиннер «Загрузка истории...», который заполняется по завершении запроса.

4. **Интегрированная фоновая вебсокет-синхронизация**:
   * При получении WebSocket-сообщения с типом `refresh` запрещено стирать локальные кэши пользователя на клиенте (`window.tasksCache`, `window.personalTasksCache`, `window.houseMembersList`). 
   * Обновление должно происходить тихо в фоне через вызов `preloadCurrentWeek().then(...)` с последующей плавной перерисовкой интерфейса.

5. **Принцип Optimistic UI (Моментальный отклик)**:
   * Любые действия пользователя с задачами (выполнение `completeHouseTask`/`completePersonalTask`, взятие себе `claimTask`, возврат в свободные `unclaimChore`, пропуск `skipChore`/`skipFreeChore`, удаление `deletePersonalTask`) должны сначала мгновенно менять состояние в локальном кэше и перерисовывать экран.
   * Сетевой запрос к бэкенду шлется в фоновом режиме с флагом `silent = true`. В случае ошибки сетевого запроса состояние UI и очков пользователя должно автоматически откатываться к исходному значению.

6. **Единый Splash Screen при старте**:
   * При запуске приложения в HTML (`index.html`) оверлей `#globalLoadingOverlay` изначально отображается без класса `hidden`. Он перекрывает интерфейс, пока идет стартовая загрузка.
   * Сквозной таймер запускается один раз в самом начале скрипта `app.js` и работает непрерывно.
   * Оверлей скрывается ровно один раз — только после завершения батч-загрузки задач и участников дома при старте.

7. **Пакетная (Batch) загрузка при запуске**:
   * Запуск Mini App делает ровно один сетевой запрос к `/api/batch/tasks`, который возвращает как задачи на 14 дней, так и участников дома (`members`). Лишние последовательные запросы к `/api/house/members` при старте запрещены.
